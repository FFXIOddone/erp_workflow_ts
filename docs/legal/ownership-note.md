# Ownership and Work-for-Hire Note

This note is a plain-language reference for the ERP workspace. It is not legal advice.

## Federal copyright baseline

The U.S. Copyright Office states that a work made by an employee within the scope of employment is
generally a work made for hire, and the employer is the author.

Source:

- `https://www.copyright.gov/register/se-hire.html`

## Practical implication

- If the code was created by employees while working within the scope of employment, the employer
  generally owns the copyright unless a different written agreement says otherwise.
- If the team wants to prevent external commercialization, sale, or reuse, that restriction needs
  to be written into a separate agreement or internal policy. A user EULA alone is not enough.
- If there are contractors or outside contributors, their rights should be covered by a separate
  assignment or license agreement.
