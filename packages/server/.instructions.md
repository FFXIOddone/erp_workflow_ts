# Server Package Instructions

When working in `packages/server/`:

## Import Convention
All imports MUST use `.js` extension — this is an ESM requirement:
```typescript
import { prisma } from '../db/client.js';
import { broadcast } from '../ws/server.js';
import { authenticate, type AuthRequest } from '../middleware/auth.js';
```

## Route Pattern
Every route file follows this structure:
```typescript
import { Router } from 'express';
import { authenticate, type AuthRequest } from '../middleware/auth.js';
import { prisma } from '../db/client.js';

export const myRouter = Router();
myRouter.use(authenticate);
```

## Required for All Routes
- `authenticate` middleware on every router (no public endpoints except health check)
- `AuthRequest` type on all route handlers for typed `req.userId` access
- Zod `.parse(req.body)` for request body validation — schemas from `@erp/shared`
- `NotFoundError`/`BadRequestError` from `../middleware/error-handler.js` for error responses
- `broadcast()` from `../ws/server.js` for any state change (create/update/delete)
- `logActivity()` from `../lib/activity-logger.js` for significant actions

## Prisma Conventions
- Import from `../db/client.js`
- Use `findUnique` + null check + `NotFoundError` instead of `findUniqueOrThrow`
- Always scope queries to authorized data when applicable
- Use `select` or `include` to avoid over-fetching
- UUIDs for all IDs

## Response Shape
```typescript
res.json({ success: true, data: result });       // Success
res.json({ success: true, data: items });         // List
res.status(201).json({ success: true, data: item }); // Created
```

## WebSocket Broadcast Shape
```typescript
broadcast({ type: 'ENTITY_ACTION', payload: data, timestamp: new Date() });
```
Types follow pattern: `ORDER_CREATED`, `ORDER_UPDATED`, `EQUIPMENT_STATUS_CHANGED`, etc.
