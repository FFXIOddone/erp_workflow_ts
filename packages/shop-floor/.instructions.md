# Shop Floor Package Instructions

When working in `packages/shop-floor/`:

## Architecture
This is a Tauri desktop app for shop floor stations. It runs natively on Windows machines at each production station.

## 6 Stations
Station components live in `src/stations/`:
- **Design** — Design department workflow
- **Printing** — Roll-to-roll, flatbed, screen print
- **Production** — General production/finishing
- **Shipping** — Shipping and receiving
- **Installation** — Field installation tracking
- **Order Entry** — New order intake

## API Communication
- Use `fetch()` with the configured API base URL for all server communication
- Do NOT use axios — Tauri apps use native fetch
- API base URL comes from the config store: `src/stores/config.ts`
- Auth token stored in: `src/stores/auth.ts`

## Tauri Native Commands
- Use `invoke()` from `@tauri-apps/api` for native operations (file dialogs, system info, etc.)
- Tauri commands are defined in `src-tauri/src/`
- Keep UI logic in TypeScript, native logic in Rust

## Real-Time Updates
- WebSocket connection managed via `src/lib/useWebSocket.ts`
- Station UIs auto-refresh when relevant WebSocket messages arrive
- Stations use `PrintingMethod` enum values for identification

## State Management
- Config store: `src/stores/config.ts` — API URL, station identity
- Auth store: `src/stores/auth.ts` — JWT token, user info
- Station-specific state: local React state within station components

## Offline Considerations
- Station apps should handle API unavailability gracefully
- Show clear connection status indicators
- Queue mutations for retry when connection is restored (where applicable)
